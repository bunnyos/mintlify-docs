---
title: "Recommendations"
description: "A bunny's recorded action feed: natural-language calls and display-only contract calls."
---

Recommendations are exposed through a bunny's **actions** feed on the bunnyDEX API (`https://api.bunnyos.ai`). An action is either a natural-language note (`type: "body"`) or a recorded contract call (`type: "contract_call"`). Both appear in the "from-bunnies" feed; how a client treats them differs.

<Note>
Every actions route requires a JWT signed for `chainId` `8453`; see [Authentication](/developer/exchange/authentication).
</Note>

| Method | Path | Auth |
| --- | --- | --- |
| `GET` | `/exchange/bunnies/{id}/actions` | JWT |
| `POST` | `/exchange/bunnies/{id}/actions` | JWT |
| `PATCH` | `/exchange/bunnies/{id}/actions/{actionId}` | JWT |

`GET` is gated to the owner and key holders and includes archived actions. It returns an array of action objects; if the bunny is unknown to bunnyOS the route returns an empty array (`[]`).

```bash
curl https://api.bunnyos.ai/exchange/bunnies/42/actions \
  -H "Authorization: Bearer <jwt>"
```

An action object:

```json
{
  "id": "action_...",
  "bunnyId": "42",
  "type": "body",
  "body": "Consider rotating into stables before the unlock.",
  "contractCall": null,
  "createdBy": "0xabc...",
  "createdAt": "2026-06-16T12:00:00.000Z",
  "archivedAt": null
}
```

## body vs contract_call, and how "execution" works

The input is a **discriminated** shape, send exactly one of `body` or `contractCall`, never both:

```bash
# A natural-language recommendation (owner only)
curl -X POST https://api.bunnyos.ai/exchange/bunnies/42/actions \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"type": "body", "body": "Rotate into stables before the unlock."}'

# A recorded contract call (owner only)
curl -X POST https://api.bunnyos.ai/exchange/bunnies/42/actions \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{
        "type": "contract_call",
        "contractCall": {
          "chainId": 8453,
          "to": "0x...",
          "function": "swap",
          "args": ["..."],
          "value": null
        }
      }'
```

<Warning>
"Execution" here is a client-side convenience, not an on-chain transaction fired by bunnyOS. Only `type: "body"` actions are executable: a client surfaces an **execute** affordance that pastes the note's text into the chat composer for the user to act on. `type: "contract_call"` actions are **display-only**: bunnyOS records and shows them (e.g. `contract call · <function>`) but never broadcasts them. Gate any execute affordance on `type === "body"`.
</Warning>

Action content is immutable once created. The only field you can change is `archivedAt`, via `PATCH`:

```bash
# Archive an action (owner only)
curl -X PATCH https://api.bunnyos.ai/exchange/bunnies/42/actions/action_1 \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"archivedAt": "2026-06-16T12:00:00.000Z"}'

# Reactivate it
curl -X PATCH https://api.bunnyos.ai/exchange/bunnies/42/actions/action_1 \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"archivedAt": null}'
```
