---
title: "Threads"
description: "Holder-gated community threads: posts, comments, votes, moderators, and bans."
---

Threads are a per-bunny forum gated to that bunny's key holders, hosted by the bunnyDEX API at `https://api.bunnyos.ai`. The owner and admins always pass the gate; a non-holder calling a thread route receives `403` (`BunnyOsForbiddenError`).

<Note>
Every thread route requires a JWT signed for `chainId` `8453` (Base mainnet); see [Authentication](/developer/exchange/authentication). There is no anonymous fallback on gated routes, so a valid session is mandatory. A `401` or `403` means the JWT is missing, expired, or unauthorized: re-run the sign-in flow.
</Note>

## Posts and comments

| Method | Path | Auth |
| --- | --- | --- |
| `GET` | `/exchange/bunnies/{id}/threads/posts` | JWT |
| `GET` | `/exchange/bunnies/{id}/threads/posts/{postId}` | JWT |
| `POST` | `/exchange/bunnies/{id}/threads/posts` | JWT |
| `DELETE` | `/exchange/bunnies/{id}/threads/posts/{postId}` | JWT |
| `GET` | `/exchange/bunnies/{id}/threads/posts/{postId}/comments` | JWT |
| `POST` | `/exchange/bunnies/{id}/threads/posts/{postId}/comments` | JWT |
| `DELETE` | `/exchange/bunnies/{id}/threads/posts/{postId}/comments/{commentId}` | JWT |

List endpoints accept optional `sort` (`new` or `top`), `limit`, and `offset` query params. The maximum `limit` is `200`; a larger value returns `400`. Each list response carries its items plus a `total` count.

```bash
# List the newest posts for a bunny
curl https://api.bunnyos.ai/exchange/bunnies/42/threads/posts?sort=new&limit=50 \
  -H "Authorization: Bearer <jwt>"
```

A post object looks like:

```json
{
  "id": "post_...",
  "bunnyId": "42",
  "title": "Q2 thesis",
  "body": "Full text here...",
  "createdBy": "0xabc...",
  "createdAt": "2026-06-16T12:00:00.000Z",
  "deleted": false,
  "deletedAt": null,
  "score": 7,
  "commentCount": 3,
  "viewerVote": "up"
}
```

`viewerVote` (`up`, `down`, or `null`) reflects the authenticated caller's own vote. Comments add `postId` and `parentCommentId` (`null` for top-level comments; set for nested replies).

Create a post or a comment with a JSON body:

```bash
# Create a post
curl -X POST https://api.bunnyos.ai/exchange/bunnies/42/threads/posts \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"title": "Q2 thesis", "body": "Here is my reasoning..."}'

# Reply to a comment (nested)
curl -X POST https://api.bunnyos.ai/exchange/bunnies/42/threads/posts/post_1/comments \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"body": "Good point.", "parentCommentId": "comment_7"}'
```

<Note>
Deleting a post or comment leaves a **tombstone**: `deleted: true`, a `deletedAt` timestamp, and blanked `title`/`body`. Reply chains are preserved, so render the tombstone rather than dropping the row.
</Note>

## Voting

| Method | Path | Auth |
| --- | --- | --- |
| `PUT` | `/exchange/bunnies/{id}/threads/posts/{postId}/vote` | JWT |
| `DELETE` | `/exchange/bunnies/{id}/threads/posts/{postId}/vote` | JWT |
| `PUT` | `/exchange/bunnies/{id}/threads/posts/{postId}/comments/{commentId}/vote` | JWT |
| `DELETE` | `/exchange/bunnies/{id}/threads/posts/{postId}/comments/{commentId}/vote` | JWT |

`PUT` takes a body of `{"direction": "up"}` or `{"direction": "down"}`. `DELETE` clears the caller's vote.

```bash
# Upvote a post
curl -X PUT https://api.bunnyos.ai/exchange/bunnies/42/threads/posts/post_1/vote \
  -H "Authorization: Bearer <jwt>" \
  -H "Content-Type: application/json" \
  -d '{"direction": "up"}'
```

<Note>
Clearing a vote (`DELETE .../vote`) returns the updated post or comment with `viewerVote` back to `null`, not a `204` or empty body. Expect the entity in the response body.
</Note>

## Moderators and bans

| Method | Path | Auth | Who |
| --- | --- | --- | --- |
| `GET` | `/exchange/bunnies/{id}/threads/moderators` | JWT | owner / admin |
| `PUT` | `/exchange/bunnies/{id}/threads/moderators/{address}` | JWT | owner |
| `DELETE` | `/exchange/bunnies/{id}/threads/moderators/{address}` | JWT | owner |
| `GET` | `/exchange/bunnies/{id}/threads/bans` | JWT | moderator / owner |
| `PUT` | `/exchange/bunnies/{id}/threads/bans/{address}` | JWT | moderator / owner |
| `DELETE` | `/exchange/bunnies/{id}/threads/bans/{address}` | JWT | moderator / owner |

Listing moderators is owner/admin-only and returns `403` otherwise. Promoting (`PUT`) and demoting (`DELETE`) a moderator is owner-only. Banning and unbanning is open to moderators and the owner.

```bash
# Promote an address to moderator (owner only)
curl -X PUT https://api.bunnyos.ai/exchange/bunnies/42/threads/moderators/0xabc... \
  -H "Authorization: Bearer <jwt>"

# Ban an address from the thread (moderator or owner)
curl -X PUT https://api.bunnyos.ai/exchange/bunnies/42/threads/bans/0xdef... \
  -H "Authorization: Bearer <jwt>"
```

Moderator and ban list responses are arrays of `{bunnyId, address, createdBy, createdAt}` entries. The `PUT` promote/ban endpoints return the created record. The `DELETE` demote/unban endpoints return `204 No Content`.

<Tip>
To check whether the signed-in wallet can view, post, or moderate a bunny's thread before you call these routes, query the capability map described in [Authentication](/developer/exchange/authentication#checking-a-wallets-permissions).
</Tip>
