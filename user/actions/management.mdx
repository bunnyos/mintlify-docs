The Actions tab has three views you switch between at the top: **inbox**, **builder**, and **history**. This page covers the maintenance side: editing actions you already have in the **builder**, and reviewing everything they have posted in **history**.

From the action list on the left, select any action to open it in the editor on the right, where you can change how it behaves, run it on demand, share it as a file, or remove it.

Actions are grouped into two sections: **Native Actions** (the starter pack that ships with `$OS`) and **Custom Actions** (the ones you create or import). Each row shows a status dot, how often it runs, how many tools it may use, and when it last ran.

<Note>
If you are new to actions, start with [Build an action](/user/actions/builder) to learn what each field does, then come back here to manage what you have.
</Note>

## Edit an action

Selecting an action opens the editor. You can change any of the following, then save:

| Field | What it controls |
| --- | --- |
| **name** | The label shown in your action list. |
| **every** | How often the action runs, picked from the dropdown. |
| **enabled / paused** | The toggle at the top right. Enabled means it runs on its schedule; paused means it stays defined but never runs automatically. |
| **instructions** | Plain-language instructions describing what to watch for, when to post an alert, and when to post a one-click recommendation. |
| **tools** | What the action is allowed to use. Toggle **all** on to let it use every available tool, or turn it off and pick specific tools. The counter shows how many are enabled. |

<Steps>
  <Step title="Open the action">
    Click an action in the list to load it into the editor.
  </Step>
  <Step title="Make your changes">
    Adjust the name, schedule, instructions, or tools.
  </Step>
  <Step title="Save">
    Click **save**. You will see a confirmation, and the list refreshes.
  </Step>
</Steps>

<Tip>
If the tools panel shows no tools available, enable some services first in [Configure, Services](/user/configure/services). Actions can only use tools you have turned on.
</Tip>

## Native actions stay locked

Native actions are protected: editing one and saving creates a **custom copy** instead of overwriting the original. The editor's save button reads **save as custom copy**, your changes land in a new editable action, and the native one stays unchanged and cannot be deleted. See [Build an action](/user/actions/builder) for the full lock rule.

## Test with Run now

Use **run now** to run an action once, on demand, without waiting for its schedule. See the Run now section of [Build an action](/user/actions/builder) for details.

## Delete a custom action

Custom actions have a **delete** button at the bottom of the editor. Deleting is permanent and removes the action from your list.

<Warning>
Deletion cannot be undone, and only custom actions can be deleted. Native actions are protected and have no delete option.
</Warning>

## Export and import actions

Actions can be saved to and loaded from files for backup or sharing, and an imported file always becomes a new custom action. See the export/import details in [Build an action](/user/actions/builder).

## Review everything in history

Switch to the **history** view to see everything your actions have ever posted: **pending**, **executed**, and **hidden**, newest first. The header shows a running tally, such as "4 pending · 12 executed · 7 hidden", and items are grouped by day under **today**, **yesterday**, and dated headers.

Two filter rows let you narrow the list:

| Filter | Options |
| --- | --- |
| **kind** | all, recommendations, alerts |
| **status** | all, pending, executed, hidden |

Each card carries its title, kind, status, time, the action that raised it, the description, any tokens mentioned, and the original execute instruction.

<Frame caption="Actions history: every signal your actions have posted, with the pending / executed / hidden tally and filters.">
  <img src="/images/app/actions-history.png" alt="The actions history view" />
</Frame>

### Restore a hidden item

When you hide a card in your [inbox](/user/actions/inbox), it is not gone: it moves to history and shows up dimmed with an **unhide** button. Click **unhide** to restore it to the live inbox.

<Tip>
History is read-only apart from **unhide**. To stop an action from posting in the first place, pause it in the editor; to remove it for good, delete it.
</Tip>
