The Bunny agent runs on an AI model, and you choose which provider and model it uses. You make this choice on the **Configure** page, in the **services** tab.

<Note>
This tab is for when you bring your own provider keys. If bunnyCompute is handling your AI for you, you do not need to set a provider here. See [Managed data or your own keys](/user/getting-started/data-and-keys).
</Note>

</Frame>

## The available providers

The **inference provider** picker lists every provider you can run on. Pick one. The default provider is **openrouter**.

| Provider | Where to get a key |
| --- | --- |
| openrouter | [openrouter.ai/keys](https://openrouter.ai/keys) |
| surplus intelligence | [surplusintelligence.ai](https://www.surplusintelligence.ai) |
| venice | [venice.ai/settings/api](https://venice.ai/settings/api) |
| economyos (virtuals protocol) | [compute.virtuals.io](https://compute.virtuals.io) |

You add each provider's key in the **api** tab, then come back here to pick the provider. See [Bring your own API keys](/user/configure/api-keys).

<Tip>
The provider list may grow over time. Whatever is listed is what BunnyOS currently supports.
</Tip>

## Choosing a provider and model

<Steps>
  <Step title="Add a key">
    In the **api** tab, add a key for the provider you want to use.
  </Step>
  <Step title="Pick the provider">
    In the **llm** tab, select your provider under **inference provider**. Only one provider is active at a time.
  </Step>
  <Step title="Pick a model">
    Open the **model** picker and choose a model (see below).
  </Step>
</Steps>

## Choosing a model

The **model** picker becomes available once the active provider has a key. Open it to see that provider's models. **Free models are listed first**, each marked with a **free** label, and paid models show their price. There is a search box at the top so you can filter a long list by name. Select a model to make it the agent's current model.

If no key is set yet for the provider, the picker prompts you to set an API key to load the model list.

## Notes per provider

<AccordionGroup>
  <Accordion title="venice and economyos">
    The Bunny agent needs a model that can use tools so it can act, not just chat. On **venice** and **economyos**, make sure the model you pick supports tool calling.
  </Accordion>
</AccordionGroup>
