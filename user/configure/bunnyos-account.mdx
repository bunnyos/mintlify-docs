title: "Connect your bunnyOS account"
description: "Connect to bunnyDEX with your wallet to unlock bunny profiles, official P&L, and API keys."

Connecting your bunnyOS account links your wallet to the bunnyOS network so the terminal can show data that lives behind a sign-in. It is entirely optional: everything you can already see while signed out keeps working exactly as before. Connecting just adds extra features on top.

## What connecting unlocks

Without a connected account you can still browse bunnyDEX, view prices and trade history, and read public profiles. Connecting adds personalized, signed-in features on top of that.

<CardGroup cols={2}>
  <Card title="Bunny profiles" icon="user-pen">
    Edit a bunny's name, bio, website, and social links (X, Discord, Telegram), and tag its methodology. Your edits follow that bunny across the whole ecosystem.
  </Card>
  <Card title="Official P&L" icon="chart-line">
    See your real cost basis plus realized, unrealized, and total profit and loss per bunny and across your whole portfolio.
  </Card>
  <Card title="Recommendations inbox" icon="inbox">
    Receive the recommendation feed from the bunnies you hold keys in or manage. Tips can be sent straight to the chat with one click.
  </Card>
  <Card title="Programmatic API keys" icon="key">
    Create API keys so your own scripts can call the bunnyDEX API directly. See [Create programmatic API keys](#create-programmatic-api-keys) below.
  </Card>
</CardGroup>

<Note>
  These features are additive. If your connection ever drops, the terminal quietly falls back to the signed-out view and shows the connect prompt again. Your experience is never blocked.
</Note>

## Connect with your wallet

You do not open a new wallet connection: the terminal reuses your existing wallet and asks you to sign a one-time message. You only ever sign a message, never a transaction.

<Steps>
  <Step title="Connect your wallet">
    You need a connected wallet first. If you haven't done this yet, see [Connect your wallet](/user/getting-started/wallet-connection).
  </Step>
  <Step title="Open the connect prompt">
    Use the **Connect to bunnyDEX** button on bunnyDEX. It also appears as a banner when a signed-in feature needs your account.
  </Step>
  <Step title="Activate your wallet if prompted">
    A brand-new wallet may need a quick one-time activation before it can sign. If so, the prompt shows an **Activate wallet** button first. Activation needs a little ETH for gas, and only the first time. Once it confirms, continue to connect.
  </Step>
  <Step title="Approve the signed message">
    Select **Connect to bunnyDEX** and approve the one-time signed message in your wallet. You only ever sign a message, never a transaction, so signing in is free.
  </Step>
  <Step title="Done">
    Your signed-in features light up. See [What connecting unlocks](#what-connecting-unlocks) above.
  </Step>
</Steps>

## Disconnect and go anonymous

To go back to signed-out access, use **Sign out** in the top bar. This signs your wallet out completely. Your signed-in features return to their signed-out state, and the public bunnyDEX view continues to work unchanged. You can reconnect at any time by signing in again.

## Create programmatic API keys

If you want to call the bunnyDEX API from your own scripts, you can create API keys once you are connected. Manage them from the bunnyOS account area, where you can create, rename, set an optional expiry, regenerate, and revoke keys.

| Action | What it does |
| --- | --- |
| **Create** | Make a new key. A name is required; expiry is optional. |
| **Regenerate** | Issue a fresh secret under the same name. The old one stops working immediately. |
| **Rename / re-expire** | Update a key's name or expiry. |
| **Revoke** | Disable a key right away. |

<Warning>
  The full secret is shown **exactly once**, when you create or regenerate a key. Copy it immediately and keep it somewhere safe. If you lose it, regenerate the key to get a new secret.
</Warning>

<Note>
  This is a quick overview. Full details for calling the API from your own code live in the [Developer authentication docs](/developer/exchange/authentication).
</Note>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Connect fails right after I approve the signed message">
    The most common cause is a brand-new wallet that needs activating. Go back and select **Activate wallet** first (see the activation step above), wait for it to confirm, then connect again.
  </Accordion>
  <Accordion title="Do I need to connect bunnyOS to trade?">
    No. Buying, selling, and browsing bunnyDEX only need a connected wallet. A bunnyOS account adds the signed-in features above on top of that.
  </Accordion>
  <Accordion title="My P&L or recommendations stopped showing">
    Your connection likely dropped. Reconnect with **Connect to bunnyDEX**, and these features come back.
  </Accordion>
</AccordionGroup>
